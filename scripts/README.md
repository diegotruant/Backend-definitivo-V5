# scripts/

Utility scripts live here as the repository evolves.

## Entry-point CLI tools (root-level, registered in pyproject.toml)

These two scripts live in the repo root because they are registered as
installable CLI entry points (`pip install -e .`):

| Script | Entry point | Purpose |
|---|---|---|
| `run_in_person_test.py` | `run-in-person-test` | Drive in-person field-testing protocols |
| `verify_package_integrity.py` | `verify-package-integrity` | Sanity-check the installed package |

## Test suites

| Suite | How to run | Purpose |
|---|---|---|
| `tests/pytest_smoke.py` | `make test` / `pytest` | CI smoke — verifies package imports correctly |
| `pytest_script_suite.py` | `python pytest_script_suite.py` | Full legacy integration suite (20+ `test_*.py` scripts) |

The `test_*.py` scripts in the root are **not** collected by pytest (by design — see
`[tool.pytest.ini_options]` in `pyproject.toml`).  Run them via `pytest_script_suite.py`
or individually with `python test_<name>.py` (requires `pip install -e ".[dev]"` first).

## Generated data / reports

`reports/` and `frontend/public/data/` are build artefacts generated by the
`analyze_*.py` scripts.  They are excluded from git (see `.gitignore`).
`synthetic_fit_dataset.tar.gz` is likewise excluded — regenerate it locally with
the dataset-generation script before running `analyze_synthetic_fit_dataset.py`.
